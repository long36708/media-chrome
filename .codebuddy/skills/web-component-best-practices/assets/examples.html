Web Component Best Practices - Examples

Basic Usage

Web Components are custom HTML elements that encapsulate their own structure, style, and behavior.

Example Counter Component

<x-counter label="My Counter"></x-counter>

Attributes

Components can be configured using HTML attributes:

<x-counter
  label="Counter with Attributes"
  value="25"
  min="0"
  max="100">
</x-counter>

JavaScript API

Components can be controlled via JavaScript:

const counter = document.querySelector('#js-counter');

counter.value = 10;
counter.label = 'Updated Label';

counter.addEventListener('valuechange', (event) => {
  console.log('New value:', event.detail.value);
});

Events

Components emit custom events for state changes:

Events: None
const counter = document.querySelector('#event-counter');

counter.addEventListener('valuechange', (event) => {
  console.log('Value changed:', event.detail.value);
});

counter.addEventListener('maxreached', () => {
  console.log('Maximum value reached');
});

counter.addEventListener('minreached', () => {
  console.log('Minimum value reached');
});

Styling

Components can be styled using CSS custom properties:

.styled-counter {
  --button-background: #ff6b6b;
  --button-hover-background: #ffa07a;
  --button-active-background: #ff4757;
  --button-color: white;
  --button-hover-border-color: #ff4757;
  --button-border-radius: 8px;
  --button-padding: 12px 20px;
  --button-font-size: 16px;
  --value-font-size: 20px;
  --value-color: #ff6b6b;
  --gap: 12px;
}

Slots

Components can accept external content through slots:

Custom Content 
<x-counter label="Counter with Slot">
  <span style="font-weight: bold;">Custom Content</span>
</x-counter>

Disabled State

Components can be disabled to prevent interaction:

<x-counter label="Disabled Counter" disabled value="25"></x-counter>

Multiple Instances

Multiple component instances work independently:

<x-counter label="Counter 1" value="10"></x-counter>
<x-counter label="Counter 2" value="20"></x-counter>
<x-counter label="Counter 3" value="30"></x-counter>

Keyboard Navigation

Components support keyboard navigation:

Try these keyboard shortcuts:

  • ↑ or ArrowUp - Increment
  • ↓ or ArrowDown - Decrement
  • Home - Set to minimum
  • End - Set to maximum
  • R - Reset

Best Practices Summary

  • ✅ Use semantic HTML for better accessibility
  • ✅ Include ARIA attributes for screen readers
  • ✅ Support keyboard navigation
  • ✅ Use CSS custom properties for theming
  • ✅ Emit custom events for state changes
  • ✅ Sync attributes with JavaScript properties
  • ✅ Clean up resources in disconnectedCallback
  • ✅ Use arrow functions for event handlers
  • ✅ Provide clear TypeScript types
  • ✅ Write comprehensive tests
